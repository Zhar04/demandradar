## Письмо 1 — goszakup.gov.kz (токен открытого API)

**Кому:** АО «Центр электронных финансов» — оператор портала госзакупок
**E-mail:** support@ecc.kz (можно продублировать через форму обратной связи на goszakup.gov.kz)
**Тема:** Запрос на доступ к унифицированным сервисам (открытому API) портала государственных закупок

> Уважаемые коллеги!
>
> Меня зовут [ФИО]. Я развиваю сервис [название, напр. «DemandRadar»] — помощник для
> предпринимателей и поставщиков Казахстана, который помогает быстрее находить
> релевантные объявления о закупках по товарным категориям и регионам и участвовать
> в подходящих тендерах.
>
> Прошу предоставить доступ к унифицированным сервисам портала государственных закупок
> (открытый API, ows.goszakup.gov.kz) и выпустить токен авторизации.
>
> Данные (объявления, лоты, справочники) планируется использовать исключительно для
> чтения и агрегации публично опубликованной информации — чтобы показывать пользователям
> подходящие тендеры. Сервис не совершает действий от имени пользователей на портале и
> не участвует в торгах: подача заявок остаётся ручным шагом самого пользователя через
> его личный кабинет.
>
> Технологический стек: Python. Обращения к API — периодические, в пределах лимитов портала.
>
> Реквизиты организации: [ИП/ТОО «___», БИН/ИИН ___].
> Контактное лицо: [ФИО, телефон, e-mail].
>
> Буду признателен за инструкцию по получению токена и условия использования сервисов.
>
> С уважением,
> [ФИО], [компания], [контакты]

**Быстрый путь (может не понадобиться письмо):** если у тебя уже есть аккаунт
организации на goszakup.gov.kz и роль «Администратор организации» — токен выпускается
самостоятельно в кабинете: **Профиль участника → Выпуск токена (для разработчиков)**.
Токен → строка `GOSZAKUP_TOKEN=` в `.env`.

## Пошагово: бесплатные ключи (можно получить самому, без писем)

### A. Telegram chat_id (для алертов)
1. В Telegram найди бота по имени, нажми **Start**, отправь любое сообщение.
2. Открой в браузере: `https://api.telegram.org/bot<ТОКЕН_БОТА>/getUpdates`
3. Найди в ответе `"chat":{"id": ЧИСЛО ...}` — это и есть chat_id.
4. Впиши в `.env`: `TELEGRAM_CHAT_ID=ЧИСЛО`.
   (Либо просто напиши боту и попроси меня — я вытащу и пропишу автоматически.)

### B. data.egov.kz — API-ключ (бесплатно, новые компании по ОКЭД)
1. Зарегистрируйся/войди на **egov.kz** (через ЭЦП или логин).
2. Перейди в **Кабинет разработчика**: https://data.egov.kz/profile/apikeylist
3. Нажми «Создать ключ» (Generate API key) — получишь строку-ключ.
4. Впиши в `.env`: `DATA_EGOV_APIKEY=<ключ>`.
   Проверить датасет вручную: https://data.egov.kz/datasets/view?index=gbd_ul

### C. Яндекс Wordstat API (бесплатно по заявке, латентный спрос)
1. Заведи/используй аккаунт **Яндекс** и зайди в **Яндекс.Директ** (direct.yandex.ru).
   Нужна минимум одна кампания — можно черновую: 1 фраза + 1 объявление, без запуска.
2. Получи **OAuth-токен**: https://oauth.yandex.ru → создай приложение с доступом к Директу.
3. Узнай свой **ClientId** приложения.
4. Напиши в поддержку Яндекс.Директа заявку на доступ к **Wordstat API** (укажи логин
   и ClientId). Доступ бесплатный, дают по заявке.
5. Токен → `.env`: `YANDEX_DIRECT_TOKEN=<токен>`.
   (Квота: ~1000 запросов/сутки — нам хватает с запасом.)

### D. hh.ru / hh.kz — токен приложения (бесплатно, вакансии watchlist)
1. Зайди на **dev.hh.ru**, войди своим аккаунтом hh.
2. **Создать приложение** → заполни название/описание («мониторинг вакансий для
   аналитики рынка»). Одобрение обычно ~несколько дней — **подай заявку заранее**.
3. После одобрения получишь `client_id` / `client_secret` → по ним берётся токен
   (client_credentials).
4. Токен → `.env`: `HH_TOKEN=<токен>`.
   (Анонимно API отдаёт 403, поэтому токен обязателен.)

### E. Живой скрапинг (mitwork, med/fms.ecc, mp.kz, RSS) — без ключей
Просто поставь в `.env`: `DR_SCRAPE_LIVE=1`. Ключи не нужны — это публичные страницы.
На первом живом прогоне сверим, что парсеры совпадают с реальной вёрсткой (могут
потребовать мелкой подстройки — это ожидаемо).
